# Triage

通过一个小型分诊角色状态机，推动项目 Issue 追踪器上的 Issue 流转。

分诊期间张贴到 Issue 追踪器上的每一条评论或 Issue **必须** 以这段声明开头：

```
> *This was generated by AI during triage.*
```

## 参考文档

- [AGENT-BRIEF.md](AGENT-BRIEF.md) — 如何写出耐用的 agent brief
- [OUT-OF-SCOPE.md](OUT-OF-SCOPE.md) — `.out-of-scope/` 知识库的运作方式

## 角色

两个 **类别** 角色：

- `bug` — 有东西坏了
- `enhancement` — 新功能或改进

五个 **状态** 角色：

- `needs-triage` — 维护者需要评估
- `needs-info` — 等待提报者提供更多信息
- `ready-for-agent` — 完全规格化，可交给 AFK agent
- `ready-for-human` — 需要人类实现
- `wontfix` — 不会处理

每个被分诊过的 Issue 应当恰好携带一个类别角色和一个状态角色。如果状态角色冲突，先标记出来并询问维护者，再做其他任何事情。

这些是规范的角色名——Issue 追踪器中使用的实际标签字符串可能不同。映射关系应当已经提供给你了——如果没有，请运行 `/setup-matt-pocock-skills`。

状态转移：未打标签的 Issue 通常先到 `needs-triage`；从那里转入 `needs-info`、`ready-for-agent`、`ready-for-human` 或 `wontfix`。`needs-info` 在提报者回复后回到 `needs-triage`。维护者可以随时覆盖——看起来异常的转移要标记出来并先询问。

## 调用

维护者调用 `/triage`，用自然语言描述他们想要什么。解读请求并行动。示例：

- "把需要我关注的都给我看看"
- "我们看看 #42"
- "把 #42 移到 ready-for-agent"
- "哪些是 agent 可以拿走干的？"

## 显示需要关注的

查询 Issue 追踪器，呈现三个分组，最旧的在前：

1. **Unlabeled** — 从未分诊过。
2. **`needs-triage`** — 评估进行中。
3. **`needs-info` 且自上次分诊笔记以来提报者有过活动** — 需要重新评估。

显示每个分组的计数和每条 Issue 的一行摘要。让维护者挑选。

## 分诊一个具体 Issue

1. **收集上下文。** 读完整 Issue（正文、评论、标签、提报者、日期）。解析之前的分诊笔记，这样你不会重复问已经解决过的问题。使用项目的领域术语表探索代码库，尊重该区域的 ADR。读 `.out-of-scope/*.md`，把任何与当前 Issue 相似的过往拒绝浮出水面。

2. **推荐。** 告诉维护者你的类别和状态推荐，附上理由，外加一段与该 Issue 相关的简短代码库摘要。等待指示。

3. **重现（仅限 bug）。** 在任何盘问之前，尝试重现：阅读提报者的步骤，跟踪相关代码，运行测试或命令。报告发生了什么——成功重现并附代码路径、未能重现，或细节不足（一个强烈的 `needs-info` 信号）。确认过的重现能产生强得多的 agent brief。

4. **盘问（如需）。** 如果 Issue 需要充实，运行一次 `/grill-with-docs` 会话。

5. **应用结果：**
   - `ready-for-agent` — 张贴一条 agent brief 评论（[AGENT-BRIEF.md](AGENT-BRIEF.md)）。
   - `ready-for-human` — 结构与 agent brief 相同，但要注明为何不能委派（判断决策、外部访问、设计决策、手动测试）。
   - `needs-info` — 张贴分诊笔记（下方模板）。
   - `wontfix`（bug） — 礼貌解释，然后关闭。
   - `wontfix`（enhancement） — 写入 `.out-of-scope/`，从评论里链过去，然后关闭（[OUT-OF-SCOPE.md](OUT-OF-SCOPE.md)）。
   - `needs-triage` — 应用该角色。如果有部分进展可加可选评论。

## 快速状态覆盖

如果维护者说"把 #42 移到 ready-for-agent"，相信他们并直接应用角色。确认你将要做的事（角色变更、评论、关闭），然后执行。跳过盘问。如果是在没有盘问会话的情况下移到 `ready-for-agent`，问他们是否想写一份 agent brief。

## Needs-info 模板

```markdown
## Triage Notes

**What we've established so far:**

- point 1
- point 2

**What we still need from you (@reporter):**

- question 1
- question 2
```

把盘问期间解决的所有内容捕获到 "established so far" 下，这样工作不会丢失。问题必须具体且可行动，而不是"请提供更多信息"。

## 恢复先前会话

如果该 Issue 上存在之前的分诊笔记，先读这些笔记，检查提报者是否回答了任何未决问题，然后呈现一张更新过的图景再继续。不要重新问已解决的问题。
